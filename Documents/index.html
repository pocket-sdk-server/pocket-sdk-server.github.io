SDK-Server接入指南 V2.0

一、概述

本文档用于指导游戏公司快速接入PocketGames手游SDK服务器端。


二、代码接入详细介绍

注意：
（ip地址：端口，appSecret）参数 请查看ReadMe.txt文件

1. Token校验接口

> 请求URL地址

http://ip地址:端口/pocketgames/client/user/verifyToken/{appId}/{userId}/{token}/{sign}

> 请求参数

HTTP协议，支持GET方法

参数名称 类型 示例值 必须 描述
appId int 10002 是 应用ID,由平台上分配
userId int 是 平台用户ID
token String 是 用户校验的token
sign String 是 参数签名{MD5(appId+userId+token+appSecret)}

> 返回参数

以JOSN格式返回结果

名称 类型 示例值 描述
code int 返回状态 200=成功,其他等于失败
error_msg String 错误信息


2. 通知充值结果接口 (CP方提供)

> 请求URL地址

由CP提供

> 请求参数

HTTP协议，支持POST方法

参数名称 类型 示例值 必须 描述
transactionID String   是   交易流水，由平台商产生，唯一
appId int 是 应用ID,由平台上分配
userId int 是 平台用户ID
roleId String 是 游戏角色ID
gameOrderId String 是 游戏订单ID
gameZoneId String 是 游戏区服ID
type int 是 0 = 游戏包充值 1= WEB充值
productName String 是 商品名称
exInfo String {“amount”100, “currency”:”USD”, “gameCoin”100,”gameCoinCurrency”:”Diamonds”} 是 额外信息
sign String 是 参数签名{MD5((transactionID+ appId + userId + gameZoneId +productName+appSecret)小写)小写}

> 返回参数

以JOSN格式返回结果

名称 类型 示例值 描述
code int 返回状态 200=成功,其他等于失败
error_msg String 错误信息


3. 根据流水号查看订单状态接口

> 请求URL地址

http://ip地址:端口/pocketgames/client/order/confirm/{appId}/{transactionId}/{sign}

> 请求参数

HTTP协议，支持GET方法

参数名称 类型 示例值 必须 描述
appId int 是 应用ID，由平台分配
transactionId String 是 交易ID
sign String 是 参数签名{MD5(appId+ transactionId+appSecret)}

> 返回参数

以JOSN格式返回结果

名称 类型 示例值 描述
code int 返回状态 200=成功,其他等于失败
data Object
data.transactionId String 交易流水，平台商产生，唯一
data.amount String 交易金额
data.currency String 交易货币
data.userId int 平台用户ID
data.time String 交易时间

由CP提供

HTTP协议，支持GET方法

三、业务流程

1. 登陆注册流程

  1. 游戏调用 SDK的登录接口
  2. 游戏拿到登录回调结果，如果 tokenInfo不为空，就按 Token流程继续处理步骤 3、4、5、6，否则就直接拿到userID跟userAccount
  3. 游戏拿到 tokenInfo传给自己的游戏服务器
  4. 游戏服务器把 tokenInfo传给平台服务器做验证
  5. 平台服务器以 JSON方式返回验证结果
  6. 游戏服务器把验证结果传回游戏中，根据 tokenCode 判断是否有效


2. 支付流程

应用客户端调用支付接口时, 需指定支付结果的通知回调地址notify_uri. 支付完成后, pocket平台服务器会把支付结果以POST方式通知到此地址. 游戏接收验证参数后, 给用户做游戏内充值。游戏服务端通知接口在接收到通知消息后, 需回应200表示通知已经接收. 如果回应其他值或者不回应, 则被认为通知失败, Pocket平台服务器会尝试多次通知. 这个机制用来避免掉单。 游戏应做好接收到多次通知的准备, 防止多次加钱. 同时, 需要特别注意的是, 回应的200表示应用已经正常接到消息, 无需继续发送通知. 它不表示订单成功与否, 或者应用处理成功与否. 对于重复的通知, 应用可能发现订单已经成功处理完毕, 无需继续处理, 也要返回200 否则, pocket平台服务器会认为未成功通知, 会继续发送通知。

  1. 游戏调用 SDK的支付接口
  2. 平台服务器返回支付结果
  3. 如果支付成功，则发送支付购买的结果到游戏服务器
  4. 游戏服务器如果收到消息，则返回OK
  5. 游戏服务器通知游戏客户端发货成功
